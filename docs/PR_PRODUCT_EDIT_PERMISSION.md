# 商品編集権限のUI側チェック

## 概要

商品編集画面でクライアント側の権限チェックを追加しました。

## 変更内容

### 変更ファイル
- `lib/presentation/screens/product_edit_screen.dart`
  - `_hasPermission` フラグを追加
  - `_checkPermission()` メソッドを追加
  - 権限がない場合はエラー表示して自動的に画面を閉じる

## 動作

1. 商品編集画面に遷移すると、現在のユーザーと商品の `creatorId` を比較
2. 一致しない場合:
   - 画面に「編集権限がありません」と表示
   - SnackBarでエラーメッセージを表示
   - 2秒後に自動的に前の画面に戻る
3. `creatorId` が `null` の場合（レガシーデータ）は誰でも編集可能

## 備考

- Firestoreセキュリティルールでもサーバーサイドで保護されているため、二重のセキュリティ対策
